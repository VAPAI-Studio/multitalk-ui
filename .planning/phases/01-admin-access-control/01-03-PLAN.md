# Plan 01-03: Frontend Auth Integration with Admin Role

## Goal
Extend AuthContext to expose isAdmin flag based on user role, and add Infrastructure studio to navigation config marked as admin-only.

## Requirements
- ADMIN-01: Admin user can access infrastructure management pages (part 1: auth integration)

## Tasks

<task>
<name>Task 1: Extend AuthContext to include role and isAdmin</name>
<action>
1. Open frontend/src/contexts/AuthContext.tsx
2. Locate the User interface (should be around line 10-20)
3. Add role field to User interface:
   ```typescript
   export interface User {
     id: string;
     email: string;
     full_name?: string;
     profile_picture_url?: string | null;
     created_at?: string;
     role?: string;  // NEW: from Supabase metadata
   }
   ```
4. Locate AuthContextType interface (around line 20-30)
5. Add isAdmin field to AuthContextType:
   ```typescript
   interface AuthContextType {
     user: User | null;
     token: string | null;
     loading: boolean;
     login: (email: string, password: string) => Promise<void>;
     register: (email: string, password: string, fullName?: string) => Promise<void>;
     logout: () => void;
     isAuthenticated: boolean;
     isAdmin: boolean;  // NEW: computed from user.role
   }
   ```
6. In AuthProvider component function (around line 40-50), add isAdmin computation using useMemo:
   ```typescript
   const isAdmin = useMemo(() => {
     return user?.role === 'admin';
   }, [user]);
   ```
7. Locate the AuthContext.Provider value prop (around line 150-160)
8. Add isAdmin to the context value object:
   ```typescript
   <AuthContext.Provider value={{
     user,
     token,
     loading,
     login,
     register,
     logout,
     isAuthenticated: !!user && !!token,
     isAdmin  // NEW
   }}>
   ```
9. Ensure verifyToken function (or wherever /auth/me is called) stores role from response:
   - Response includes userData.role from backend
   - setUser should include role: userData.role
10. Result: AuthContext now exposes isAdmin boolean derived from user.role
</action>
