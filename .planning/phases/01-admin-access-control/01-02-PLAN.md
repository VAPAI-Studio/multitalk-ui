# Plan 01-02: Infrastructure Router and API Protection

## Goal
Create admin-protected infrastructure API router, register it in main app, and document admin role implementation.

## Requirements
- ADMIN-04: Backend API endpoints enforce admin-only access with 403 responses for non-admins

## Tasks

<task>
<name>Task 1: Create infrastructure API router with admin protection</name>
<action>
1. Create new file: backend/api/infrastructure.py
2. Add standard imports:
   ```python
   from fastapi import APIRouter, Depends
   from typing import Dict, Any
   from core.auth import verify_admin
   ```
3. Create router with prefix and tags:
   ```python
   router = APIRouter(prefix="/api/infrastructure", tags=["infrastructure"])
   ```
4. Add placeholder health check endpoint (will be used in Phase 2):
   ```python
   @router.get("/health")
   async def infrastructure_health(
       admin_user: dict = Depends(verify_admin)  # Per-endpoint protection
   ) -> Dict[str, Any]:
       """
       Infrastructure health check endpoint.
       Admin-only: Returns basic status.

       NOTE: This project uses per-endpoint protection, not router-level dependencies.
       All future endpoints added to this router must explicitly include
       Depends(verify_admin) in their signature.
       """
       return {
           "success": True,
           "message": "Infrastructure API available",
           "admin_user_id": admin_user.id
       }
   ```
5. Result: New admin-protected API namespace ready for future infrastructure endpoints

Key security point:
- Depends(verify_admin) automatically returns 403 for non-admins
- verify_admin already exists in backend/core/auth.py (no implementation needed)
- Protection is per-endpoint: each endpoint must include Depends(verify_admin) explicitly
- This pattern ensures future endpoints don't accidentally skip admin checks
</action>
