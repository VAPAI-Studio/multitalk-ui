**VariablesStep sub-component:**

Props: `{ state: BuilderState; onUpdate: (partial: Partial<BuilderState>) => void; onNext: () => void; onBack: () => void; setStatus: (s: string) => void; isLoading: boolean; setIsLoading: (v: boolean) => void }`

State: `dragIndexRef = useRef<number | null>(null)`, `const [isDragging, setIsDragging] = useState(false)`

Handlers:
- `handleDragStart(e, index)`: `dragIndexRef.current = index; e.dataTransfer.effectAllowed = 'move'; setIsDragging(true)`
- `handleDrop(e, dropIndex)`: reorder variableConfig array, update `order` field for each, call `onUpdate({ variableConfig: reordered })`
- `handleDragEnd()`: `dragIndexRef.current = null; setIsDragging(false)`
- `handleUpdateVariable(id, partial)`: update matching variable in variableConfig
- `handleRemoveVariable(id)`: filter out variable, renumber `order`

Render:
- Header: "Configure Variables ({state.variableConfig.length})"
- If empty: show info box "Go back to the Inspect step to add variables from workflow nodes."
- List of VariableCard components with drag handlers
- Navigation buttons: "Back" → onBack, "Next: Dependencies" → async save then onNext
  - On Next: `setIsLoading(true)`, call `apiClient.updateCustomWorkflow(state.workflowId!, { variable_config: state.variableConfig as Record<string, unknown>[], section_config: state.sectionConfig as Record<string, unknown>[] })`, `setIsLoading(false)`, `onNext()`
