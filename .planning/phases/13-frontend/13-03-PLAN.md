Add a "Batch History" section to the BatchVideoUpscale page, shown below the main content when the user is in the upload view (not monitoring an active batch).

**1. Batch history state and loading:**
```typescript
const [batchHistory, setBatchHistory] = useState<any[]>([]);
const [historyLoading, setHistoryLoading] = useState(false);
```

Add a `loadHistory` function that calls `apiClient.listUpscaleBatches()` and sets `batchHistory`. Call it:
- On component mount (in a useEffect)
- After a batch completes (when polling detects terminal state)
- When user clicks "New Batch" to return to upload view

**2. BatchHistoryCard sub-component:**
Each batch renders as a card with:
- Status badge (reuse StatusBadge from Task 2 of Plan 02)
- Creation date formatted as "Mar 11, 2026 at 3:45 PM" (use `new Date(batch.created_at).toLocaleDateString('en-US', { month: 'short', day: 'numeric', year: 'numeric' })` + `toLocaleTimeString`)
- Video counts: "X videos (Y completed, Z failed)"
- Settings summary: resolution, flavor (e.g., "2k, vivid")
- Progress bar (if processing)
- Two action buttons:
  - "View" -- loads the batch detail and switches to monitoring view (sets `activeBatchId` to this batch ID)
  - "Re-run with same settings" -- pre-fills the settings form and switches to upload view

**3. Re-run flow (handleRerun function):**
When user clicks "Re-run with same settings":
1. Read settings from the batch: `{ resolution, creativity, sharpen, grain, fps_boost, flavor }`
2. Set `settings` state to these values
3. Clear `activeBatchId` (return to upload view)
4. Show a toast/notice: "Settings loaded from previous batch. Upload new videos to start."
5. Scroll to the upload section
The user must upload NEW videos -- the original videos' files are not re-downloaded. Per research recommendation: "re-run" means new batch with same settings, user uploads new videos.

**4. "View" flow:**
When user clicks "View" on a history card:
1. Set `activeBatchId = batch.id`
2. The existing polling useEffect (from Plan 02) will automatically start fetching the batch detail

**5. History section layout:**
```tsx
{!activeBatchId && (
  <Section title="Batch History">
    {historyLoading ? (
      <p className="text-gray-500 dark:text-gray-400">Loading history...</p>
    ) : batchHistory.length === 0 ? (
      <p className="text-gray-500 dark:text-gray-400">No previous batches. Upload videos above to get started.</p>
    ) : (
      <div className="space-y-4">
        {batchHistory.map(batch => (
          <BatchHistoryCard key={batch.id} batch={batch} onView={...} onRerun={...} />
        ))}
      </div>
    )}
  </Section>
)}
```

**6. Dark mode classes on all elements.** Follow the existing pattern in the component from Plan 02.

**7. Edge cases:**
- If history API returns an error, show "Failed to load history" with a retry button
- If batch history is very long (>20), show only the 20 most recent with a "Show more" button (simple pagination: increment a `historyLimit` state by 20)
