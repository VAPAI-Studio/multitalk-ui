  <action>
Add to `CustomWorkflowService`:

```python
async def execute_dynamic_workflow(
    self,
    workflow_config: dict,       # Row from custom_workflows table
    user_params: dict,           # User-provided parameter values
    base_url: str,               # ComfyUI server URL
    client_id: str,              # WebSocket client ID
) -> Tuple[bool, Optional[str], Optional[str]]:
```

Implementation:
1. Extract slug from workflow_config
2. Build template name as `custom/{slug}` (WorkflowService._find_template_path searches subdirectories)
3. Call `self.workflow_service.build_workflow(template_name, user_params)` -- this loads the template from `backend/workflows/custom/{slug}.json`, substitutes `{{PLACEHOLDER}}` values, and checks for unsubstituted placeholders
4. If build fails, return (False, None, error)
5. Call `self.workflow_service.validate_workflow(workflow)` -- checks node structure
6. If validation fails, return (False, None, error)
7. Create `ComfyUIService()` instance
8. Call `comfyui_service.submit_prompt(base_url, {"prompt": workflow, "client_id": client_id})`
9. Return the result tuple

This is intentionally a thin orchestrator. The real work happens in existing services. This satisfies TEST-04: both the test runner and renderer call this exact function.

Add unit tests using `unittest.mock.patch` and `AsyncMock` to mock `WorkflowService.build_workflow`, `WorkflowService.validate_workflow`, and `ComfyUIService.submit_prompt`. Test all three failure paths and the success path. Verify the exact arguments passed to each service call.
