    - detect_workflow_format returns "unknown" for empty dict or unrecognized structure
    - detect_workflow_format skips keys starting with "_" (like _meta)
    - is_link_input returns True for [string, int] arrays like ["14", 0]
    - is_link_input returns False for regular values (strings, numbers, bools, other arrays)
    - parse_workflow_nodes extracts node_id, class_type, title (from _meta.title), all inputs
    - parse_workflow_nodes marks link inputs with is_link=True
    - parse_workflow_nodes populates configurable_inputs with only non-link inputs
    - parse_workflow rejects UI format with clear error message about Dev Mode and "Save (API Format)"
    - parse_workflow rejects unknown format with error message
    - parse_workflow returns success with format="api" and structured nodes for valid API-format JSON
  </behavior>
  <action>
Create `backend/services/custom_workflow_service.py` with a `CustomWorkflowService` class. For now, implement ONLY the parsing-related methods (CRUD and execute come in Plan 02):

1. `detect_workflow_format(data: dict) -> str` -- Static method. Returns "api", "ui", or "unknown". UI format: has top-level "nodes" (list) or "links" (list) or "version" (int/float). API format: at least one top-level value (ignoring "_"-prefixed keys) is a dict with "class_type". Unknown: neither pattern matches.

2. `is_link_input(value) -> bool` -- Static method. Returns True if value is a list of exactly 2 elements where [0] is str and [1] is int (e.g., ["14", 0]).

3. `parse_workflow_nodes(workflow_json: dict) -> List[ParsedNode]` -- Iterates all top-level keys (skipping "_"-prefixed). For each node: extracts node_id (the key), class_type, title from `_meta.title` if present. For each input: creates ParsedNodeInput with is_link set via is_link_input. Populates configurable_inputs as the subset where is_link=False.

4. `async parse_workflow(workflow_json: dict) -> Tuple[bool, Optional[ParseWorkflowResponse], Optional[str]]` -- Orchestrator: detects format, rejects UI/unknown, parses nodes, returns ParseWorkflowResponse.

For the UI format rejection message, use: "This appears to be a UI-format workflow. Please enable Dev Mode in ComfyUI settings and use 'Save (API Format)' to export."

Also include `__init__` with `self.supabase = supabase or get_supabase()` and `self.workflow_service = WorkflowService()` for later use by CRUD/execute methods (Plan 02). Import from `core.supabase` and `services.workflow_service`.
