  <files>backend/api/custom_workflows.py, backend/main.py</files>
  <action>
Create `backend/api/custom_workflows.py` with a FastAPI `APIRouter(prefix="/api/custom-workflows", tags=["custom-workflows"])`. Follow the exact pattern from `backend/api/infrastructure.py` -- per-endpoint `Depends(verify_admin)`, NOT router-level dependencies.

Endpoints:

1. `POST /parse` -- Parse a workflow JSON
   - Request: `ParseWorkflowRequest`
   - Admin: `Depends(verify_admin)`
   - Instantiate `CustomWorkflowService()`, call `parse_workflow(payload.workflow_json)`
   - Return `ParseWorkflowResponse` (200 for both success and parse failure -- use response.success field)

2. `POST /` -- Create a custom workflow
   - Request: `CreateCustomWorkflowRequest`
   - Admin: `Depends(verify_admin)`
   - Call `service.create(payload, created_by=admin_user.id)`
   - Return `CustomWorkflowResponse` with 201 on success, 400 on failure
   - On slug conflict (error contains "duplicate" or "unique"), return 409

3. `GET /` -- List all custom workflows
   - Admin: `Depends(verify_admin)`
   - Call `service.list_all()`
   - Return `CustomWorkflowListResponse`

4. `GET /published` -- List published workflows only
   - Admin: `Depends(verify_admin)`
   - Call `service.list_published()`
   - Return `CustomWorkflowListResponse`

5. `GET /{workflow_id}` -- Get single workflow by ID
   - Admin: `Depends(verify_admin)`
   - Call `service.get(workflow_id)`
   - Return `CustomWorkflowResponse`, 404 if not found

6. `PUT /{workflow_id}` -- Update a workflow
   - Request: `UpdateCustomWorkflowRequest`
   - Admin: `Depends(verify_admin)`
   - Call `service.update(workflow_id, payload)`
   - Return `CustomWorkflowResponse`

7. `DELETE /{workflow_id}` -- Delete a workflow
   - Admin: `Depends(verify_admin)`
   - Call `service.delete(workflow_id)`
   - Return `{"success": True}` or raise 404/500

8. `POST /{workflow_id}/publish` -- Publish a workflow
   - Admin: `Depends(verify_admin)`
   - Call `service.toggle_publish(workflow_id, True)`
   - Return `CustomWorkflowResponse`

9. `POST /{workflow_id}/unpublish` -- Unpublish a workflow
   - Admin: `Depends(verify_admin)`
   - Call `service.toggle_publish(workflow_id, False)`
   - Return `CustomWorkflowResponse`

IMPORTANT: Use `admin_user: dict = Depends(verify_admin)` on EVERY endpoint. Do NOT use router-level dependencies.

- Add: `app.include_router(custom_workflows.router)` (no prefix -- the router already has `/api/custom-workflows` prefix)
