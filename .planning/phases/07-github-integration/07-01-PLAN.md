<objective>
Add GitHub release creation to trigger RunPod rebuilds after Dockerfile commits.

Purpose: RunPod monitors GitHub releases (not branch pushes) to trigger automated Docker image builds. The existing save_dockerfile endpoint commits and pushes via the Contents API, but does not create a release. This plan adds the release creation capability as an opt-in deploy trigger.

Output: Extended GitHubService with create_release() method, updated DockerfileSaveRequest model with trigger_deploy field, extended save_dockerfile endpoint that optionally creates a release after commit, and comprehensive tests.

    - Import `datetime` at top of file (from datetime import datetime)
    - After the successful `service.update_file()` call (line that extracts `commit_sha = result["commit"]["sha"]`), add the deploy trigger block:

    ```python
    commit_sha = result["commit"]["sha"]

    # Optionally create GitHub release to trigger RunPod rebuild
    release_info = None
    deploy_error = None
    if payload.trigger_deploy:
        from datetime import datetime
        timestamp = datetime.utcnow().strftime("%Y%m%d-%H%M%S")
        tag_name = f"deploy-{timestamp}"
        try:
            release_result = await service.create_release(
                tag_name=tag_name,
                target_commitish=commit_sha,
                name=f"Deploy {timestamp}",
                body=payload.commit_message.strip(),
            )
            release_info = {
                "tag_name": release_result["tag_name"],
                "html_url": release_result.get("html_url", ""),
            }
        except httpx.HTTPStatusError as release_err:
            deploy_error = f"Commit succeeded but release creation failed: {release_err.response.text}"

    return {
        "success": True,
        "commit_sha": commit_sha,
        "deploy_triggered": bool(payload.trigger_deploy and release_info),
        "release": release_info,
        "deploy_error": deploy_error,
    }
    ```

    - The existing 409/error handling in the outer try/except remains unchanged
    - Note: The return dict now always includes `deploy_triggered`, `release`, and `deploy_error` keys (backward compatible since frontend currently only reads `success` and `commit_sha`)
