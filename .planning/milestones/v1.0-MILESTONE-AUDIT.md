## Integration Findings

### Finding 1: VOL-04 Pagination Not Wired in UI
**Severity:** Gap (partial requirement)
**File:** `frontend/src/components/FileTree.tsx:46`
**Detail:** Backend returns `hasMore: boolean` and `continuationToken: string|null` but FileTree ignores them. Directories with 201+ items silently show only first 200, with no "Load more" control.
**Affects:** VOL-04

### Finding 2: Double API Call on File Operations
**Severity:** Tech debt (performance)
**File:** `frontend/src/components/FileTree.tsx:57-60`
**Detail:** `FileTree.handleRefresh()` calls both `loadDirectory()` AND `onRefreshRequest?.()`. The latter increments `refreshTrigger` in Infrastructure.tsx, which remounts FileTree and triggers another `loadDirectory()`. Result: 2× GET /files calls per delete/rename/move.
**Affects:** FILEOP-01 through FILEOP-06 (functionally correct, minor inefficiency)

### Finding 3: Infrastructure Router Self-Prefix Pattern
**Severity:** Tech debt (maintainability)
**Files:** `backend/api/infrastructure.py:35`, `backend/main.py:41`
**Detail:** All other routers rely on `main.py` adding `prefix="/api"` at registration. `infrastructure.py` defines its own full prefix `/api/infrastructure` and is registered with no prefix in `main.py`. Routes resolve correctly but pattern is inconsistent and is a maintenance trap.
**Affects:** No runtime issue for v1.0
