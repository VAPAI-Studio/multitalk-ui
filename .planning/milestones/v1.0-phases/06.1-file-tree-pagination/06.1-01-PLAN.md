<objective>
Close the VOL-04 gap: add "Load more" pagination controls to FileTree and FileTreeNode, and fix the double API call on file operations by replacing the key-based remount pattern with a refreshId prop.

Purpose: Directories with more than 200 items silently truncate today. Admins managing large model volumes (10,000+ files) cannot see all content. Every file operation also fires two identical GET /files API calls due to the double-trigger in handleRefresh + Infrastructure.tsx key remount.

Output: FileTree.tsx and FileTreeNode.tsx with full append-on-load-more pagination. Infrastructure.tsx without key remount. Five new test cases confirming the behaviors. Zero backend changes needed (backend already supports pagination).
