## Decisions Made

- **refreshId prop over key= remount:** The key= remount pattern destroys all component state (expanded folder nodes collapse). The refreshId prop triggers only a data reload via useEffect, preserving tree state.
- **Double-call root cause:** The original handleRefresh called both loadDirectory() directly AND onRefreshRequest?.() which incremented refreshTrigger which changed the key= which triggered a remount which fired the mount useEffect which called loadDirectory() again. Fix: remove onRefreshRequest?.() from handleRefresh.
- **jsdom downgrade (deviation fix):** jsdom 27 introduced @asamuzakjp/css-color which uses require(@csstools/css-calc) — an ESM-only module — causing ERR_REQUIRE_ESM in all vitest runs. Downgrading to jsdom 25 resolves the conflict. This is a pre-existing blocker noted in STATE.md Pending Todos.
